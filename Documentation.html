OrganizationServiceData

The OrganizationService relates to a service such as an A360 account, and the OrganizationServiceData thus contains information about elements in the basic hierarchy (such as folders and files) of each project within the organization.

The class relates to data for four service types in the project hierarchy: projects (additional data to the normal Project type), folders, items (models, documents and other files) and versions (of the items).

Each object will contain information about parent objects under the 'relationships' key. For folders, the parent will be a project. For items, the parent will be a folder. For versions, the parent will be an item.

CustomMetadata

CustomMetadata composes the custom properties from Vitus that expand upon the original project metadata composed by properties of original models from various native software. The CustomMetadata thus enables properties such as links between many types of objects, e.g. between a Wall element in a specific and a PDF document in the project. Additionally it adds progess properties, allowing users to add and specify progress categories and stages for model elements. 


              {
                "1601fdb5-9a60-4c4b-8b62-46ff255d82a5": {
                  "documents": {
                    "links": {

Create Session

The following snippet will start a session. Since the api uses cookies, a cookie container is needed. 


              DefaultApi api = new DefaultApi();

              VitusAPI.Client.Configuration.Default.ApiClient.RestClient.CookieContainer = new CookieContainer();

              Login login = new Login('USERNAME', 'PASSWORD');
              api.CreateSession(login);

After login, the cookie is used to store information about user. This mean endpoints like: 


              Profile userProfile = api.GetProfile();

can be called without defining the user.

Access Project

After creating a session, the accessable Organizatins and Projects can now be found.

Step 1: Get Organization

All data is based on Organization | Project | File Therefore first step is done by: 


              Organization vitusOrganization = api.GetUserOrganization('ORGANIZATION-NAME');

Step 2: Get Project List

With the use of the an Organization ID, a list of Projects to which the users has access can then be obtained with 


              List<Project> projectList = api.GetProjects(vitusOrganization.Id)

Getting file info

To get Artefacts or CustomMetaData you first need to obtain the OrganizationServiceData object for the file. This can be done in 2 ways

Method 1: Search for filename

Searching for a filename will give you a list of all OrganizationServiceData Items that contains the searched name. 


              List<OrganizationServiceData> searchOutput = api.SearchProjectData(vitusOrganization.Id, project.Id, 'SEARCHTERM');

After getting the item, each version are then obtained by:


              OrganizationServiceData dataById = api.GetProjectDataById(vitusOrganization.Id, project.Id, fileItem.Id);

The versions will be accesible in under the Content key


              List<OrganizationServiceData> fileVersions = dataById.Content;

Method 2: List all project data

Listing all project data will give you a full object with all folders and files. (DOES NOT WORK!! / NRR) 


              OrganizationServiceData projectData = api.GetProjectData(vitusOrganization.Id, project.Id);

The data will be accesible in under the Content key


              List<OrganizationServiceData> projectDataItem = projectData.Content;

Getting CustomMetaData

CustomMetadata are stored on items such as a project, folders, files and versions. Therefore getting any custom metadata from files is done by 


              var customMetadata = api.GetCustomMetadata(vitusOrganization.Id, fileItem.Id);

The CustomMetadata output will be include both links and progress data for each element in the file.

Posting CustomMetaData

CustomMetadata are stored on files items as JSON formattet data.

Adding Links and Progress should be done with 2 different methods.

Method 1: Progress


              var customMetadata = api.AddCustomMetadataVersion(vitusOrganization.Id, fileItem.Id, 'CHANGESET');

Method 2: Links


              var customMetadata = api.AddCustomMetadata(vitusOrganization.Id, fileItem.Id, 'CHANGESET');

Getting object info

Object info are stored on file versions. Therefore getting any object data from files is done by 


              ModelArtefact fileArtefact = api.GetProjectDataArtefact(vitusOrganization.Id, project.Id, fileVersion.Id, "indexes");

The ModelArtefact will include properties found aftr analysis of model on Vitus.
